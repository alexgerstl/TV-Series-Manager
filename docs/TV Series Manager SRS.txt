1. Introduction
1.1 Purpose

TV Series Manager is a Windows desktop application designed to automate and simplify the management of locally stored television series.

The application focuses on the following workflow:

Download new episodes.
Convert videos if necessary.
Organize videos into the local library.
Handle subtitle files.
Detect embedded subtitles.
Extract subtitles if desired.
Check online for newer episodes.
Synchronize the local library with a NAS.
Provide one-click access to frequently used tools.

Unlike applications such as Sonarr, this application never performs destructive operations automatically. Every file movement, deletion, or synchronization must be initiated by the user.

1.2 Goals

The application shall

organize TV series
minimize repetitive manual work
provide a modern UI
remain responsive during long-running tasks
be modular and extensible
be production-ready
support future integrations
1.3 Non Goals

The application is not intended to

download torrents
automatically rename downloaded releases
automatically move files without user interaction
replace Plex or Jellyfin
automatically synchronize without confirmation

# 15. MKVToolNix Integration

## Overview

The application shall integrate the MKVToolNix suite to inspect MKV containers and extract embedded subtitle tracks.

The integration shall be optional. If MKVToolNix is not installed or configured, all other application features shall continue functioning normally.

All interaction with MKVToolNix shall be encapsulated in a dedicated service named `MKVToolNixService`.

No other module shall invoke MKVToolNix executables directly.

---

# 15.1 Supported Executables

The application shall support the following executables.

| Executable | Required | Purpose |
|------------|----------|---------|
| mkvmerge.exe | Yes | Inspect container metadata |
| mkvextract.exe | Yes | Extract subtitle tracks |
| mkvinfo.exe | Optional | Display detailed container information |
| mkvpropedit.exe | Optional | Future metadata editing |

Default installation directory

```text
C:\Program Files\MKVToolNix\
```

The executable paths shall be configurable and persisted in SQLite.

The project shall be considered complete when:

✓ All configured folders are monitored correctly.

✓ Files are processed only after explicit user interaction.

✓ Converted videos are preferred over incoming videos.

✓ Embedded English subtitles are detected and displayed.

✓ English subtitles can be extracted individually or in batches.

✓ MKV metadata is cached and refreshed correctly.

✓ Web lookup correctly identifies newer episodes.

✓ New episodes are highlighted and counted.

✓ Search results can be opened in the default browser.

✓ NAS availability is monitored continuously.

✓ Network drives can be mounted and unmounted.

✓ Local vs NAS comparison only shows missing files.

✓ Copy and Move operations verify successful transfers before deleting local files.

✓ External applications launch correctly.

✓ Workflow shortcuts open the correct folders.

✓ All settings persist between application restarts.

✓ Errors are handled gracefully and logged.

✓ The UI remains responsive during long-running operations.

✓ The architecture is modular, testable, and extensible.

✓ The application is suitable for long-term maintenance and future feature expansion.
